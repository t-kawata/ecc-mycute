# JPush — 日本語コミット＆プッシュ

前回のコミット以降の変更を `git diff` / `git status` で分析し、変更内容を日本語で説明するコミットメッセージを生成して `make push msg="..."` を自動実行する。

## Process

### Step 1: 変更状態を確認

```bash
git status --short
git diff --stat
git log --oneline -5
```

`git status --short` が空の場合は「変更はありません」と報告して終了。

### Step 2: 変更内容を詳細に分析

```bash
git diff
git diff --cached
git status
```

出力を読み、以下の観点で分類する：
- 新規追加されたファイル・機能
- 修正された既存コード
- 削除されたファイル
- 設定や依存関係の変更
- テストやドキュメントの変更

### Step 3: 日本語コミットメッセージを生成

分類結果をもとに、以下のフォーマットで日本語のコミットメッセージを生成する：

```
<prefix>: <日本語での変更概要>
```

**prefix の種類**（conventional commits に準拠）:

| prefix | 使用場面 |
|--------|---------|
| `feat` | 新機能・新規ファイル追加 |
| `fix` | バグ修正 |
| `refactor` | 動作変更を伴わないコード整理 |
| `docs` | ドキュメントのみの変更 |
| `test` | テストの追加・修正 |
| `chore` | ビルド・設定・依存関係 |
| `perf` | パフォーマンス改善 |
| `style` | フォーマット・スタイルのみの変更 |

**メッセージ作成ルール**:
- 先頭は prefix + `:` + スペース（例: `feat: `）
- 変更内容を日本語で簡潔に（1行72文字以内が理想。やむを得ない場合は80文字程度まで）
- 単なる作業ログではなく、「何ができるようになったか」「何が直ったか」を記述
- 複数の種類の変更が混在する場合は、主要な変更をprefixに選び、補足をメッセージに含める

例:
```
feat: ログアウトボタンを追加
fix: ユーザー名に特殊文字を含む場合の認証エラーを修正
chore: SeaORM を v0.12 にアップグレード
refactor: セッション管理を共通モジュールに抽出
```

### Step 4: make push を実行

```bash
make push msg="<生成したコミットメッセージ>"
```

### Step 5: 結果報告

実行結果を報告する。失敗した場合はエラー内容を表示して終了。

## Edge Cases

- **`make push` が定義されていない**: Makefile が存在しない、または `push` ターゲットがない場合。「Makefile に push ターゲットが見つかりません」と報告して終了。
- **変更量が膨大**: diff の全出力が巨大な場合、`git diff --stat` を優先し、詳細は主要ファイルのみ確認する。
- **コンフリクト残留**: `git status` に `UU` や `AA` などのコンフリクトマーカーがある場合、「コンフリクトが解消されていません」と報告して終了。
